# 第三章：結局場景佈置與重置按鈕視覺設計

當玩家成功揭開《Winter House》命案的真相後，我們需要給他們一個充滿儀式感的收尾。在這一章中，我們將建立遊戲的最後一站：**End_Scene (結局場景)**，並把「重新開始」的視覺介面準備好，為後續的程式串接打下基礎。

---

## 🖼️ 第一步：佈置 End_Scene 的沉浸式畫面

結局場景的佈置邏輯與 Start_Scene 非常相似，我們同樣依賴 Canvas 與 2D UI 來呈現最後的故事。

1. 在 `Project` 視窗的 `_Scenes` 資料夾中，點擊右鍵建立並打開名為 **`End_Scene`** 的新場景。
2. 建立 `Canvas`，並**務必記得**將右側的 `Canvas Scaler` 改為 `Scale With Screen Size` (輸入 1920x1080)，避免玩家在看結局時遭遇破壞氣氛的跑版慘劇（詳見第二章教學）。
3. **加入背景圖：** 在 Canvas 下新增一個 `Image`，你可以放入一張 AI 生成的「大雪紛飛的木屋窗外」或是「破曉時分的別墅」作為破關底圖，並將大小拉滿整個畫面。
4. **加入結局文本：** 新增 `Text (TextMeshPro)`，輸入結局的描述文字，例如：*「你發現了杯底的粉末。原來這場讀書會，從一開始就是為他設下的死亡陷阱...」*。別忘了套用我們在第一章做好的「動態中文字體 (Dynamic)」，讓文字完美顯示。

---

## 🔄 第二步：建立「重新開始」按鈕外觀

除了觀看結局，我們還必須給玩家一個回到遊戲起點的按鈕介面。

1. 在 Canvas 下建立一個 **`Button - TextMeshPro`**。
2. 展開按鈕，將底下的文字修改為 **「重新開始」** 或 **「再次調查」**，並同樣套用動態中文字體。
3. **設定錨點 (Anchor)：** 依照第二章教過的防跑版技巧，你可以把這個按鈕的錨點釘死在畫面的「正下方」或「右下角」，確保它在任何螢幕上都會乖乖待在對的位置。
4. **美化按鈕：** 你可以在 Button 的 `Image` 元件中，換上符合《Winter House》冷冽風格的 UI 按鈕圖片，或者調整顏色的透明度，讓它完美融入背景。

---

## 🏗️ 階段性結語：分離視覺與邏輯的優雅架構

到目前為止，我們已經完成了《Winter House》三個核心場景（Start、Play、End）的所有「視覺與實體佈置」。

**為什麼我們現在還不寫按鈕的腳本？**
因為在遊戲開發中，將「UI 視覺設計」與「程式邏輯控制」分階段處理，是最清晰的工作流。現在我們的舞台已經完全搭建完畢，UI 也不會跑版，中文字體也完美顯示。

在接下來的章節中，我們將正式進入「核心機制實戰篇」，開始撰寫 C# 腳本。到時候，我們不僅會教你怎麼讓這個「重新開始」按鈕發揮作用，還會透過這三個獨立佈置好的場景，展現出「完全不需要寫重置程式碼，就能達成完美進度歸零」的超強架構！
