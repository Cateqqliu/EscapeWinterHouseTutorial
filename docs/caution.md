# 番外篇：《Winter House》Unity 開發避坑總結與除錯指南

以下是我在開發完成後請和我協作的AI幫我整理出來的內容：

在獨立開發《Winter House》的過程中，我們走過了 UI 排版、腳本串接、音效實裝，直到最終的 Windows 打包出廠。

對於剛接觸 Unity 的開發者來說，有幾個「經典陷阱」幾乎是每個人必踩的。這篇番外篇專門為你整理了純 Unity 開發環境中最容易犯的錯誤清單。如果你的遊戲突然罷工了，請先來這裡對症下藥！

---

## 🖼️ 一、 UI 介面與點擊陷阱

### 1. 明明是按鈕，滑鼠點了卻毫無反應
* **症狀：** 遊戲畫面的「開始遊戲」或「關閉」按鈕，滑鼠游標移上去完全不會變色，點下去也毫無反應。
* **病因：** 建立 Canvas 時，Unity 會自動生成一個 `EventSystem` 物件，你不小心把它刪掉了！它是所有 UI 點擊事件的靈魂。
* **解法：** 在左側 `Hierarchy` 空白處按右鍵 ➔ `UI` ➔ **`Event System`**，把它召喚回來，按鈕就會瞬間復活。

### 2. 實體道具點不到，好像有一道「隱形牆」
* **症狀：** 馬克杯的 Box Collider 加了，腳本也寫了，但是怎麼點都沒反應。
* **病因：** 通常是因為你的 UI 畫布（例如滿版透明的對話框背景）擋在 3D/2D 場景的最前面，把滑鼠的點擊訊號給「攔截」了。
* **解法：** 檢查 Canvas 裡那些「不需要被點擊」的裝飾或透明背景，在右側的 `Image` 元件中，將 **`Raycast Target`** (射線偵測目標) **取消勾選**。

### 3. 中文字變成打叉方塊 `[X] [X]`
* **症狀：** 在 UI 按鈕或對話框輸入中文後，畫面只顯示方塊。
* **病因：** TextMeshPro (TMP) 預設不認識繁體中文。
* **解法：** 匯入中文字型並建立 Font Asset 後，務必將 `Generation Settings` 裡的模式改為 **`Dynamic`（動態）**，按下 Apply，並確實套用到 UI 文字上。

### 4. 畫面一放大，對話框就跑到螢幕外面
* **症狀：** 打包成全螢幕的 `.exe` 後，UI 比例完全失控。
* **病因：** Canvas 預設為絕對像素，不會跟著玩家的螢幕放大。
* **解法：** 建立 Canvas 後的第一件事：把右側 `Canvas Scaler` 改為 **`Scale With Screen Size`**（如 1920x1080），並確實釘好 UI 的錨點（Anchor）。

## ⚙️ 二、 GameManager 與邏輯陷阱

### 5. 按鈕綁定腳本時，拉了 C# 檔案卻找不到功能 (🌟 新手超常踩！)
* **症狀：** 在設定 Button 的 `On Click()` 時，把寫好的 C# 腳本直接拖進去，旁邊的下拉選單卻找不到 `StartGame()` 或其他寫好的功能。
* **病因：** Unity 的按鈕無法直接讀取專案資料夾裡的「腳本檔案 (`.cs`)」。它需要的是一個活在場景中、掛載了該腳本的「實體物件 (GameObject)」。
* **解法：** 請先確保你的腳本已經掛載到左側 `Hierarchy` 的某個空物件上（例如 `--- Managers ---`）。在設定按鈕時，**要拖曳的是那個 `--- Managers ---` 物件**，而不是底下的腳本檔案！拖進去後，下拉選單就會乖乖出現腳本名稱與功能了。

### 6. 點擊道具後爆出 `NullReferenceException` 紅字
* **症狀：** 點了馬克杯，主控台噴出可怕的 `NullReferenceException: Object reference not set to an instance of an object`。
* **病因：** 腳本寫好了，但你「忘記接線」了！大總管 GameManager 找不到你要它打開的對話框到底在哪裡。
* **解法：** 點選掛著 `GameManager` 的空物件，看右側的 `Inspector`。確保你有把 Canvas 底下的 `Dialogue Panel` 等 UI 物件，**確實拖曳到腳本對應的空欄位裡**。

### 7. 按下開始按鈕，主控台報錯說找不到場景
* **症狀：** 出現 `Scene couldn't be loaded...` 的錯誤訊息。
* **病因：** Unity 不認識沒有註冊過的場景。程式碼沒錯，是名單漏了。
* **解法：** 打開 `File` ➔ `Build Settings`，把所有場景（Start, Play, End）都拖進 `Scenes In Build` 裡，並確保 Start_Scene 排在最上面（編號 `0`）。

## 🔊 三、 音效與播放陷阱

### 8. 遊戲一打開，所有音效同時炸裂
* **症狀：** 進入 Play 場景的瞬間，開門聲、玻璃碰撞聲全部同時響起一次，嚇人一跳。
* **病因：** 掛在 GameManager 身上負責播放短促音效的喇叭（Audio Source），不小心勾選了自動播放。
* **解法：** 點選負責播放 SFX 的 `Audio Source`，**務必取消勾選 `Play On Awake`**。只有 BGM（背景音樂）的喇叭才需要勾選它。

---

## 📦 四、 專案管理與打包陷阱

### 9. 想備份專案，檔案卻大到好幾 GB
* **症狀：** 想把專案打包成 ZIP，卻發現容量大得誇張。
* **病因：** 壓縮了整個 Unity 資料夾，包含裝滿暫存檔的 `Library`。
* **解法：** 備份原始碼時，**只保留 `Assets`、`Packages`、`ProjectSettings` 這三個資料夾**。

### 10. 傳給朋友的遊戲打不開！
* **症狀：** 朋友點開你傳的 `Winter House.exe` 卻只看到黑畫面或閃退。
* **病因：** 你只傳了 `.exe`，沒有連同旁邊的資料夾一起給對方。
* **解法：** 打包出廠後，必須把 `.exe` 與旁邊的 `_Data` 資料夾、`MonoBleedingEdge` 資料夾，**全部圈選起來包成一個 ZIP 壓縮檔**，再傳給玩家！

---

> **💡 開發者心法：** > 遇到 Bug 不用慌張，善用 Unity 左下角的 `Console`（主控台），它閃爍的紅字通常會準確告訴你是哪個腳本、第幾行出了錯。祝你在獨立開發的路上，享受解謎與創造的樂趣！
